## 數據解讀(主公看這段)

- **每筆交易 fixed 扣 0.585% 成本**(來回手續費+稅),加滑價在價格內(≈ 0.1% 來回)。
- 5% 目標的短線:扣完剩 ~4.3%,**目標被吃掉 14%** — 不算微小。
- 真正受傷的是「勝率 50% 上下、報酬 0.5-1% 的策略」 — 成本可能把整個 edge 吃光,需要重新評估是否值得執行。
- 高 win rate 大跌幅策略(e.g. taiex_alpha)受影響相對小,因為 0.6% 成本對 5%+ 的平均報酬只佔 ~12%。

## 後續行動

1. **重訓 ML model 時** target/label 若用 `apply_costs=True` 的版本,整體門檻會升高 → 預期 picks 數量略減、但「實際進得了場」的 picks 底氣更強。
2. **Streamlit 顯示**:預設仍是 apply_costs=True(主公規格);若要看「無成本上限」做 sanity check,程式碼裡 apply_costs=False 開回去即可。
3. **參數調優**:若主公實際券商有折扣(e.g. 28 折),回測時可傳 `broker_fee_discount=0.28`,成本會降到 ~0.38%。
